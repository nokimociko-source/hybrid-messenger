# Исправление: Аватар группы не отображается после сохранения

## Проблема
После сохранения аватара группы в GroupSettingsModal, изменения не отображались без перезагрузки страницы.

## Решение
Добавлена realtime подписка на изменения комнаты в хуке `useSupabaseRoom`, чтобы автоматически обновлять данные при изменении в базе данных.

## Изменённые файлы

### 1. `hybrid_messenger/client/src/app/hooks/useSupabaseChat.ts`
- Добавлена realtime подписка в `useSupabaseRoom` на события UPDATE таблицы `rooms`
- Теперь при изменении данных комнаты (включая avatar_url) компонент автоматически обновляется

### 2. `hybrid_messenger/client/src/app/pages/client/CatloverProfilePanel.tsx`
- Упрощён callback `onUpdate` в GroupSettingsModal
- Удалён `window.location.reload()` - теперь обновление происходит автоматически через realtime

## Как это работает

1. Пользователь сохраняет новый аватар в GroupSettingsModal
2. Данные обновляются в таблице `rooms` в Supabase
3. Realtime подписка в `useSupabaseRoom` получает событие UPDATE
4. Автоматически вызывается `fetchRoom()` для получения свежих данных
5. Компонент перерисовывается с новым аватаром

## Преимущества
- Нет необходимости в перезагрузке страницы
- Изменения видны мгновенно
- Работает для всех пользователей в реальном времени
- Меньше кода и проще поддержка

## Проверка
После применения изменений:
1. Откройте настройки группы
2. Загрузите новый аватар
3. Сохраните изменения
4. Аватар должен обновиться сразу без перезагрузки
