# Применение обновлений: Бан участников и Настройки группы

## Обзор

Этот документ описывает, как применить обновления для:
1. **Бан участников** - добавление полей для бана в таблицу `room_members`
2. **Настройки группы** - UI для изменения названия и аватара группы

## 1. Миграция базы данных

### Применение SQL миграции

Выполните SQL скрипт для добавления полей бана:

```bash
psql -U your_user -d your_database -f add_ban_fields_to_room_members.sql
```

Или через Supabase Dashboard:
1. Откройте SQL Editor
2. Скопируйте содержимое `add_ban_fields_to_room_members.sql`
3. Выполните запрос

### Что добавляется

В таблицу `room_members` добавляются поля:
- `is_banned` (BOOLEAN) - флаг бана
- `banned_at` (TIMESTAMPTZ) - дата бана
- `banned_by` (UUID) - кто забанил
- `ban_reason` (TEXT) - причина бана
- `ban_expires_at` (TIMESTAMPTZ) - дата окончания бана (NULL = навсегда)

- Только для админов и создателей

## 3. Функционал бана

### Как забанить участника

1. Откройте профиль группы
2. Перейдите на вкладку "Участники"
3. Нажмите на три точки рядом с участником
4. Выберите "Забанить"

### Что происходит при бане

- Участник помечается как забаненный (`is_banned = true`)
- Сохраняется информация о бане (кто, когда, причина)
- Участник остается в таблице `room_members`, но не может отправлять сообщения
- Можно установить срок бана или забанить навсегда

### Разбан участника

Для разбана нужно обновить запись:

```sql
UPDATE room_members
SET is_banned = FALSE,
    banned_at = NULL,
    banned_by = NULL,
    ban_reason = NULL,
    ban_expires_at = NULL
WHERE id = 'member_id';
```

## 4. Функционал настроек группы

### Как изменить настройки группы

1. Откройте профиль группы
2. Нажмите на три точки (меню настроек)
3. Выберите "Настройки группы"
4. Измените название, описание или аватар
5. Нажмите "Сохранить"

### Права доступа

- Только админы и создатели могут изменять настройки группы
- Обычные участники не видят эту опцию

## 5. Проверка работы

### Тест бана

```sql
-- Проверить забаненных участников
SELECT 
    rm.id,
    u.username,
    rm.is_banned,
    rm.banned_at,
    rm.ban_reason
FROM room_members rm
JOIN users u ON u.id = rm.user_id
WHERE rm.room_id = 'your_room_id'
AND rm.is_banned = TRUE;
```

### Тест настроек группы

```sql
-- Проверить обновление группы
SELECT id, name, topic, avatar_url
FROM rooms
WHERE id = 'your_room_id';
```

## 6. Автоматический разбан

Функция `auto_unban_expired()` автоматически разбанивает участников по истечении срока.

Для автоматического запуска можно настроить cron job:

```sql
-- Создать расширение pg_cron (если еще не создано)
CREATE EXTENSION IF NOT EXISTS pg_cron;

-- Запускать каждый час
SELECT cron.schedule(
    'auto-unban',
    '0 * * * *',
    'SELECT auto_unban_expired();'
);
```

## Troubleshooting

### Ошибка: "User is banned from this room"

Это нормально - триггер предотвращает вход забаненных пользователей.

### Не сохраняется аватар группы

Проверьте права доступа к storage bucket `media`:
- Должна быть включена публичная загрузка
- RLS политики должны разрешать загрузку админам

### Не видно кнопки "Настройки группы"

Проверьте:
- Вы админ или создатель группы?
- Это групповой чат (не личный)?
