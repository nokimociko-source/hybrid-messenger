# 🚀 E2E Шифрование - Быстрый старт

**Время:** ~15 минут  
**Сложность:** Легко

---

## 📋 Шаг 1: Применить миграцию БД (5 минут)

1. Откройте **Supabase Dashboard**
2. Перейдите в **SQL Editor**
3. Откройте файл: `database/migrations/enable_zero_knowledge_e2e_step_by_step.md`
4. Копируйте и выполняйте каждый SQL блок **по очереди** (17 шагов)

**Важно:** Выполняйте по одному блоку! Supabase не может выполнить несколько запросов сразу.

**Проверка:**
```sql
SELECT table_name FROM information_schema.tables 
WHERE table_name IN ('room_keys', 'e2e_audit_log');
```

Должно вернуть 2 таблицы ✅

---

## 📋 Шаг 2: Добавить UI настроек (2 минуты)

Добавьте кнопку в настройки профиля:

**Файл:** `client/src/app/components/SettingsModal.tsx` (или где у вас настройки)

```typescript
import { E2ESettings } from './E2ESettings';
import { useState } from 'react';

function YourSettingsComponent() {
  const [showE2E, setShowE2E] = useState(false);

  return (
    <>
      <button onClick={() => setShowE2E(true)}>
        🔒 Настройки шифрования
      </button>

      {showE2E && (
        <E2ESettings onClose={() => setShowE2E(false)} />
      )}
    </>
  );
}
```

## 📋 Шаг 3: Тестирование (5 минут)

### 3.1 Включить E2E

1. Откройте приложение
2. Откройте настройки → "Настройки шифрования"
3. Включите переключатель "Включить E2E шифрование"
4. Нажмите "Сохранить"

### 3.2 Отправить сообщение

1. Откройте чат с другим пользователем
2. Отправьте сообщение: "Тест E2E"
3. Сообщение должно отправиться

### 3.3 Проверить в БД

Откройте Supabase → Table Editor → messages → последнее сообщение:

```
content: "🔒{\"ciphertext\":\"...base64...\",\"iv\":\"...\",\"ephemeralPublicKey\":\"...\"}"
```

✅ Если видите такой формат - шифрование работает!

### 3.4 Проверить расшифровку

1. Обновите страницу
2. Откройте тот же чат
3. Сообщение должно отображаться расшифрованным: "Тест E2E"

✅ Если видите расшифрованное сообщение - всё работает!

## 🐛 Проблемы?

### Сообщения не шифруются

1. Проверьте, что E2E включено в настройках
2. Проверьте localStorage: `localStorage.getItem('e2e_enabled')` должно быть `"true"`
3. Проверьте, что у получателя есть публичный ключ в БД

### Не могу расшифровать сообщения

1. Проверьте, что ключи сохранены: `localStorage.getItem('ecdh_keys')`
2. Попробуйте перегенерировать ключи
3. Проверьте консоль браузера на ошибки

### Ошибка в SQL миграции

1. Убедитесь, что выполняете по одному блоку
2. Проверьте, что таблицы `users`, `rooms`, `messages` существуют
3. Проверьте, что расширение `uuid-ossp` включено
