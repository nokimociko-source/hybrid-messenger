# 🔒 Руководство по E2E шифрованию

## Обзор

Сквозное шифрование (End-to-End Encryption, E2E) обеспечивает максимальную приватность ваших сообщений. Когда E2E включено, сообщения шифруются на вашем устройстве перед отправкой и могут быть расшифрованы только получателем.

## Как включить E2E

1. Откройте профиль любого чата (кнопка с информацией в правом верхнем углу)
2. Нажмите кнопку **"🔒 Шифрование E2E"**
3. В открывшемся окне переключите тумблер **"Включить E2E шифрование"**
4. Нажмите **"Сохранить"**

## Как это работает

### Технические детали

- **Алгоритм:** RSA-2048 с SHA-256
- **Генерация ключей:** Автоматическая при первом запуске
- **Хранение:**
  - Приватный ключ: localStorage (только на вашем устройстве)
  - Публичный ключ: Supabase (доступен для шифрования сообщений)
- **Маркировка:** Зашифрованные сообщения помечены префиксом 🔒

### Процесс шифрования

1. Вы пишете сообщение
2. Приложение проверяет, включено ли E2E
3. Если включено:
   - Сообщение шифруется публичным ключом получателя
   - Зашифрованный текст отправляется на сервер
   - Добавляется префикс 🔒 для визуальной индикации
4. Получатель расшифровывает сообщение своим приватным ключом

## Управление ключами

### Просмотр публичного ключа

В окне настроек E2E вы можете увидеть превью вашего публичного ключа (первые и последние 20 символов). Полный ключ хранится в базе данных.

### Перегенерация ключей

⚠️ **Внимание:** Перегенерация ключей сделает все старые зашифрованные сообщения нечитаемыми!

Когда нужно перегенерировать:
- Если вы подозреваете компрометацию ключа
- При смене устройства
- Для повышения безопасности (раз в год)

Как перегенерировать:
1. Откройте настройки E2E
2. Прокрутите вниз до раздела "Перегенерировать ключи"
3. Нажмите кнопку **"Перегенерировать"**
4. Подтвердите действие

## Ограничения текущей реализации

### ⚠️ Важно знать

1. **Упрощенная реализация:** Текущая версия использует базовое RSA шифрование. Для production рекомендуется Signal Protocol или Matrix Olm.

2. **Групповые чаты:** В групповых чатах сообщение шифруется только для первого участника. Полная реализация требует шифрования для каждого участника отдельно.

3. **Медиа-файлы:** Файлы и изображения пока не шифруются, только текстовые сообщения.

4. **Синхронизация устройств:** Ключи не синхронизируются между устройствами. Каждое устройство имеет свою пару ключей.

5. **Восстановление:** Если вы потеряете приватный ключ (очистка localStorage), старые зашифрованные сообщения станут нечитаемыми.

## Безопасность

### Что защищено

✅ Содержимое текстовых сообщений
✅ Приватный ключ (хранится только на устройстве)
✅ Защита от перехвата на сервере

### Что НЕ защищено

❌ Метаданные (кто, кому, когда отправил)
❌ Медиа-файлы
❌ Информация о комнатах и участниках
❌ Реакции и другие действия

## Рекомендации

### Для максимальной безопасности

1. **Включайте E2E только в приватных чатах** — в групповых чатах текущая реализация не обеспечивает полную защиту

2. **Регулярно обновляйте ключи** — раз в год перегенерируйте ключи

3. **Не делитесь приватным ключом** — никогда не экспортируйте и не передавайте приватный ключ

4. **Используйте сильные пароли** — для защиты аккаунта

5. **Будьте осторожны с устройствами** — не оставляйте разблокированное устройство без присмотра

## FAQ

**Q: Могу ли я использовать E2E в групповых чатах?**
A: Технически да, но текущая реализация шифрует только для одного участника. Полная поддержка групп будет добавлена позже.

**Q: Что произойдет, если я очищу кеш браузера?**
A: Вы потеряете приватный ключ и не сможете расшифровать старые сообщения. Публичный ключ останется в БД.

**Q: Можно ли восстановить зашифрованные сообщения?**
A: Нет, если вы потеряли приватный ключ. Это фундаментальное свойство E2E шифрования.

**Q: Видит ли сервер мои сообщения?**
A: Сервер видит зашифрованный текст, но не может его расшифровать без приватного ключа.

**Q: Безопасно ли это для конфиденциальной информации?**
A: Текущая реализация подходит для базовой защиты, но для критически важной информации рекомендуется дождаться интеграции Signal Protocol.

## Поддержка

Если у вас возникли проблемы с E2E шифрованием:

1. Проверьте консоль браузера на наличие ошибок
2. Попробуйте перегенерировать ключи
3. Убедитесь, что localStorage не заблокирован
4. Создайте issue в репозитории проекта
